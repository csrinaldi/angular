Compilación


Permite Agregar al nueva sintaxis, es decir nuevos elementos u/o atributos



El proceso de compilación ocurre en dos fases:



  • Compile: Recorre el DOM buscando directivas, el resultado es una función de linking
  • Link: Combina las directivas con un scope y produce una vista viva, cualquier cambio en el modelo (scope) es reflejado en la vista y viceversa.

Que es una Directiva?

Podemos decir que una directiva es una marca en el DOM que sera utilizada por el compilador de AngularJS para anexa algun comportamiento especial a ese elemento, o que transforme el Elemento o sus Hijos.


Que significa compilar?

Para AngularJS compilar siginifica atachar "event listeners" al HTML para hacerlo interactivo.

Controllers

Los controladores es una funcion que anexa valores y/o comportamiento al scope asociado. Este escope, es tambien manejado por la vista



var app = angular.module('app', []);
    app.controller('FirstController', function($scope) {
    $scope.counter = 0;
    $scope.add = function(value){
        $scope.count = $scope.count + value;
    }
    $scope.subtract = function(value){
        $scope.count = $scope.count - value;
    }
});

<div ng-controller="FirstController">
    <h4>The simplest adding machine ever</h4>
    <button ng-click="add(1)" class="button">Add</button>
    <a ng-click="subtract(1)" class="button alert">Subtract</a>
    <h4>Current count: {{ counter }}</h4>
</div>

Expresiones

Angular Expressions son parecidas a las Expressions de JavaScript pero con algunas diferencias, y suelen colocarlse entre "{{}}"


{{1+5}}


{{store.products}}

Diferencias con JavaScript Expressions


Contexto: Las expresiones de JS son evaluadas en el contexto de "windows", las expresiones de Angular son evaluadas en el contexto del scope


Forgiving: Las expresiones de JS cuando es evaluado algo no definido, se genera ReferenceError o TypeError, en Angular el resultado es undefined o null


No Control Flow Statements: En AngularJS no se puede utilizar condicionales, loops o excepciones.


Filters: Es posible utilizar Filtros de Angular en las Expreciones para manejar la presentacion.

Data Binding

Data-binding es una sincronización automática de datos entre la vista y el modelo. La forma en que Angular implementa el data-binding permite tratar al modelo como single-source-of-truth en su aplicación. La vista es una proyección de la modelo en todo momento. Cuando el modelo cambia, la vista refleja el cambio, y viceversa.


Scopes

Es un objeto hace referencia al modelo en la aplicación.
El scope es un contexto de ejecución para las expresiones.
Ademas, los scopes son organizados en una estructura jerárquica imitando la estructura del DOM.
Puden ver las expresiones y propagar eventos

  • Los scopes proveen un método $watch en su API para observar los cambios en el modelo.
  • Proveen un método $apply para propagar los cambios a través del sistema desde la vista hacia afuera (Controllers, Services, Angular event handlers.
  • Los scopes pueden ser nested, limitando asi el acceso a propidades del modelo:
    • Child Scope: Hereda las propiedades del padre
    • Insolated Scope: No hereda las propiedades del padre

El scope es un pegamento entre la vista y el controller

En la fase de linking del template la directiva setea $watch expressions en el scope.
$watch permite a la directiva ser notificada de los cambios en las propiedades, lo que permite actualizar el DOM.

Tanto controller como la directiva tienen una referencia al scope, pero no entre si

Este principio de visibilidad es muy óptimo porque hace asnóstico de la vista al controller y favorece las pruebas


function MyController($scope) {
    $scope.username = 'World';
    $scope.sayHello = function() {
        $scope.greeting = 'Hello ' + $scope.username + '!';
    };
}

Shadow DOM

  • shadow root: Nuevo tipo de nodo
  • shadow host: Es todo nodo que tiene asociado un shadow root, El contenido del shadow host no es renderizado, en su lugar es renderizado el contenido del shadow root
                                
                                    
<button>Hello, world!</button>
<script>
    var host = document.querySelector('button');
    var root = host.createShadowRoot();
    root.textContent = 'こんにちは、影の世界!';
</script>

                                
                            
                                
<script>
    var host  = document.querySelector('button');
    var label = document.getElementById("content_shadow_host");
    label.value = host.textContent;
</script>

Polymer

Polymer, esta construido sobre las últimas tecnologías para el desarrollo de aplicaciones modernas, pero nativamente no todo está soportado, por lo tanto, la capa mas baja de Polymer es platform.js o polyfill que son un conjunto de librerías que emulan lo que actualmente no esta soportado

platform.js esta desarrollado para ser utilizado de manera separa, por ejemplo: x-tag utiliza un subconjunto de platform.js.

  • Web Components
    • Shadow DOM. Encapsulate DOM subtrees and the associated CSS.
    • HTML Imports. Load element definitions and other resources declaratively.
    • Custom Elements . Define new elements in HTML.
  • DOM
    • URL. Parse URLs in JavaScript.
    • WeakMap. Shim for ES6 WeakMap type.
    • Mutation Observers. Efficiently watch for changes in the DOM.
    • observe-js. Observe changes on JS objects/arrays using Object.observe (if available).
  • Other
    • Pointer Events. Handle both touch and mouse seamlessly
    • Pointer Gestures. More complex touch gestures.
    • Web Animations. Define complex timeline animations.
